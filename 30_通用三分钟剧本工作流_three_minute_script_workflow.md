# 30_通用三分钟剧本工作流

更新时间：2026-06-21
版本：v2｜通用剧本版

## 文件定位

本文件是 C01 视频创作 / 剧本类项目的通用三分钟剧本生产工作流。

适用范围：

```text
短剧
漫剧
AI 长剧
真人短剧
动画短剧
电影片段
系列短视频剧情
1-3 分钟单集 / 单段剧情
```

核心目标：

```text
把一个剧情点做成可观看、可拍摄、可剪辑、可持续连载的三分钟剧本单元。
```

本工作流不绑定任何具体项目、人物、世界观、平台或题材。具体项目使用时，应先替换为本项目的主角、敌人、世界规则、平台目标、视觉资产和禁项。

## 01 启动输入

每次启动三分钟剧本工作流，先确认：

```text

目标观众：
发布平台：

本集所在位置：第几集 / 第几章 / 第几个片段。
上一集最后一帧：
本集必须承接的信息：
本集禁止提前揭示的信息：
已有主角 / 反派 / 场景 / 道具规则：
```

### 1.1 输入不足自动补全

如果用户只给一句话、一个剧情点或一个模糊想法，不要停住等用户填完整表。先自动补齐最低可执行信息：

```text
主角是谁：
主角此刻想要什么：
阻力是谁 / 是什么：
不做会损失什么：
本集可见危机：
本集小答案：
结尾更大追问：
```

自动补全只能作为初稿假设；如果补全内容涉及项目正典、人物关系、世界规则或平台定位，必须标注“暂定”，后续可被用户修正。

## 02 题材适配开关

进入三分钟结构前，先判断题材重点。不同题材用同一套 180 秒骨架，但优化重点不同。

| 题材 | 三分钟重点 | 特别检查 |
|---|---|---|
| 动作 / 战斗 | 空间压迫、动作接触点、结果落地 | 敌人不能站着等打；动作必须有起点、接触点、结果 |
| 悬疑 / 反转 | 信息差、误导、证据、结尾追问 | 反转必须有前置线索，不能靠硬藏信息 |
| 爱情 / 情感 | 欲望冲突、误会、选择、情绪递进 | 不能只靠表白，要靠行动和选择推动关系 |
| 喜剧 | 预期反差、节奏、角色反应 | 笑点不能破坏人物动机和剧情目标 |
| 恐怖 / 惊悚 | 未知压力、声音、空间、逼近感 | 不要过早解释怪物规则；恐惧要逐步逼近 |
| 奇幻 / 科幻 | 世界规则、能力限制、视觉奇观 | 不能为爽点临时加能力；奇观必须服务剧情 |
| 商战 / 权谋 | 利益目标、筹码、交换、背叛 | 每个反转要改变资源或权力关系 |
| 成长 / 逆袭 | 当前弱点、阶段小胜、代价 | 不能一步登天；成长必须有损耗或选择 |

如果题材混合，选择一个主开关和一个副开关，不要每种题材规则都塞进去。

## 03 总原则

```text
1. 三分钟只讲一个清楚的小目标。
2. 开场先给画面和危机，不慢解释。
3. 主角必须在压力下主动选择。
4. 每 20-40 秒必须有一次信息变化、压力升级、动作选择或情绪转折。
5. 本集解决一个小问题，同时打开一个更大的问题。
6. 不能为了爽点破坏角色能力、世界规则、战力边界和长线因果。
7. 剧本要能被拍、能被剪、能被转成分镜和视频提示词。
8. 如果是连续剧，单集必须承接上一集，并给下一集留下明确入口。
```

## 04 三分钟标准结构

| 时间段 | 功能 | 具体要求 |
|---|---|---|
| 0-15 秒 | 开场钩子 / 留存起点 | 给强画面、强危机、强问题，观众立刻知道“出事了” |
| 15-45 秒 | 本集目标 | 主角必须做什么，不做会失去什么 |
| 45-105 秒 | 冲突推进 | 敌人规则、环境阻碍、资源不足、人物误判、压力升级 |
| 105-145 秒 | 选择 / 反转 / 代价 | 主角做出选择，小胜或受创，不能靠免费外挂 |
| 145-170 秒 | 小答案 / 小爽 / 弧光 | 回收本集最小问题，推进一点人物、关系或谜题 |
| 170-180 秒 | 结尾钩子 / 下一集点击 | 打开下一集更具体、更危险、更想知道的追问 |

如果不是完整单集，而是三分钟片段，也按这六段结构设计：开场抓人、目标明确、压力升级、选择代价、小落点、尾钩。

## 07 三视角创作复盘优化闸

插入位置：

```text
单集生产卡 -> 正式剧本正文初稿 -> 三视角创作复盘优化 -> 改出优化稿 -> 连续性检查 -> 最终审稿闸门。
```

这个环节不是只挑错，而是把剧本从“能讲清楚”优化到“更好看、更能留人、更能拍、更能剪”。

### 7.1 顶级编剧视角

逐集检查并优化：

```text
1. 本集小目标是否足够具体。
2. 主角是否在压力下主动选择，而不是被剧情推着走。
3. 情绪弹簧是否一层一层压紧。
4. 开场问题、中段转折、小答案、结尾追问是否形成因果链。
5. 台词能否改成动作、表情、道具或选择。
6. 本集是否留下观众想争论、想猜、想追的点。
```

### 7.2 顶级导演视角

逐集检查并优化：

```text
1. 前 3 秒首帧是否一眼有危机、奇观或反常。
2. 每个时间段是否有明确可拍画面，不是抽象解释。
3. 人物站位、压力来源、出入口、危险点是否清楚。
4. 动作起点、接触点、结果和落点是否能拍出来。
5. 表演是否具体到眼神、呼吸、失衡、停顿、狠意、恐惧或犹豫。
6. 本集是否至少有一个可被记住的视觉画面。
```

### 7.3 顶级剪辑视角

逐集检查并优化：

```text
1. 0-15 秒是否能留住人，是否过早解释。
2. 每 20-40 秒是否有一次信息变化、压力升级、选择动作或画面变化。
3. 180 秒内是否有节奏起伏：压迫、短停、爆发、反转、落点。
4. 转场是否能用动作、声音、道具、视线或空间方向自然接上。
5. 是否有可以剪成强预告、强封面、强片段的镜头。
6. 结尾是否在最高追问点切断，而不是把情绪说完。
```

### 7.4 三视角输出格式

```text
编剧优化点：
导演优化点：
剪辑优化点：
必须改动的剧本位置：
优化后单集稿：
仍需进入审稿闸门的问题：
```

如果三视角复盘发现开场弱、目标散、动作不可拍、节奏平、结尾不想追，不能直接进入最终审稿，必须先改稿。

## 08 连续性检查

如果是连续剧、系列剧、长篇短剧或多集项目，三视角优化后必须检查连续性：

```text
上一集留下什么：
本集开场兑现了什么：
本集新增了什么问题 / 资源 / 伤势 / 关系 / 敌人收益：
本集解决了什么小问题：
本集不能解决什么大问题：
下一集必须承接什么：
```

连续性硬规则：

```text
1. 上一集的伤势、道具、位置、敌人压力不能消失。
2. 本集新增的能力、资源、线索必须有来源。
3. 本集结尾钩子下一集必须能接，不得假钩子。
4. 长线谜题只能推进，不得无代价直接揭完。
5. 如果本集是独立短片，也要检查开头信息是否足够观众理解。
```

## 09 三分钟剧本审稿闸门

```text
1. 0-15 秒是否有可见问题，而不是慢慢解释。
2. 本集小目标是否一句话说清。
3. 主角是否在压力下做选择，而不是被剧情推着走。
4. 本集是否只解决一个小问题。
5. 本集是否打开一个更大的问题。
6. 台词是否过多，能否动作化。
7. 能力、资源、战力或信息是否跳级。
8. 伏笔是否可见、可听、可拍。
9. 结尾是否强到想看下一集。
10. 是否能继续转成分镜、视频提示词或拍摄执行稿。
11. 是否有复看隐藏点或评论触发点。
12. 是否没有硬转化、说教感、资料堆砌感。
```

## 10 通过判定与退回路径

| 结论 | 标准 | 下一步 |
|---|---|---|
| 通过 | 结构完整、开场强、目标清楚、动作可拍、结尾可追、无连续性硬伤 | 输出用户需要的版本 |
| 条件通过 | 主体成立，但有小问题可在正文中直接修补 | 列出补丁并输出修正版 |
| 退回重写 | 开场弱、目标散、主角不主动、动作不可拍、节奏平、结尾不想追、连续性断裂任一出现 | 回到对应环节重写 |

退回路径：

```text
开场弱 -> 退回 0-15 秒开场钩子。
目标散 -> 退回本集一句话和本集小目标。
压力弱 -> 退回本集主压力和 45-105 秒冲突推进。
主角被动 -> 退回 105-145 秒选择 / 反转 / 代价。
动作不可拍 -> 退回导演视角和正文画面动作。
节奏平 -> 退回剪辑视角和 20-40 秒节奏点。
结尾弱 -> 退回 170-180 秒结尾钩子。
连续性断 -> 退回上一集承接 / 下一集入口。
```

## 11 礼部TV / LibTV 十五秒版本化承接

本环节不是旧 15 秒工作流；旧版本已清空。本环节只规定三分钟剧本工作流和后续礼部TV完整提示词之间的正确承接顺序。

触发条件：

```text
1. 用户要求把三分钟剧本做成礼部TV版本、LibTV版本、AI视频提示词版本。
2. 用户要求按 15 秒一段拆分。
3. 用户要求把三分钟剧本继续变成可生成视频的完整提示词。
```

正确顺序：

```text
三分钟剧本母稿
-> 顶级编剧 / 顶级导演 / 顶级剪辑三视角优化
-> 三分钟剧本通过判定
-> 礼部TV / LibTV 十五秒版本化
-> 每 15 秒一段，共 12 段
-> 每段再进入逻辑性 / 连续性 / 因果关系审核
-> 按审核结果回修礼部TV十五秒提示词
-> 输出最终可生成视频版本
```

### 11.1 十五秒拆分原则

三分钟默认拆成 12 段：

| 段落 | 时间 | 功能 |
|---|---|---|
| 01 | 00:00-00:15 | 首帧钩子 / 危机建立 |
| 02 | 00:15-00:30 | 目标显形 / 压力加深 |
| 03 | 00:30-00:45 | 行动启动 / 第一阻力 |
| 04 | 00:45-01:00 | 冲突升级 |
| 05 | 01:00-01:15 | 空间 / 敌人 / 资源变化 |
| 06 | 01:15-01:30 | 中段转折 |
| 07 | 01:30-01:45 | 主角选择 |
| 08 | 01:45-02:00 | 选择代价 |
| 09 | 02:00-02:15 | 小胜 / 小败 / 反打 |
| 10 | 02:15-02:30 | 结果落地 |
| 11 | 02:30-02:45 | 小答案 / 情绪落点 |
| 12 | 02:45-03:00 | 下一集钩子 |

每段 15 秒只承担一个清晰画面任务，不要把两个以上复杂动作塞进同一段。

### 11.2 每段三视角处理

每个 15 秒段落都必须同时过三个视角：

```text
顶级编剧：
本段推进什么剧情？制造什么压力？给出什么信息？隐藏什么信息？让观众期待下一段什么？

顶级导演：
首帧是什么？人物站位在哪里？镜头从哪里看？动作如何开始？接触点在哪里？最后画面停在哪里？

顶级剪辑：
本段 15 秒内节奏怎么走？哪里短停？哪里爆发？上一段最后一帧怎么接？下一段第一帧怎么接？
```

### 11.4 后置审核位置

逻辑性、连续性、因果关系审核放在礼部TV十五秒版本化之后。

审核对象不是三分钟母稿，而是已经拆好的 12 段十五秒提示词：

```text
1. 段与段首尾帧是否接上。
2. 人物位置、伤势、道具、服装是否连续。
3. 动作是否有起点、接触点、结果和落点。
4. 每个结果是否有前因。
5. 每个反转是否有铺垫。
6. 每个钩子下一段 / 下一集是否能兑现。
```

审完以后必须回修礼部TV十五秒提示词本身，不只输出审核报告。

## 12 输出模式选择

根据用户需求，选择一种输出：

| 用户需求 | 默认输出 |
|---|---|
| 想看故事是否成立 | 三分钟剧情框架 |
| 想直接阅读 | 三分钟正式剧本正文 |
| 想优化已有稿 | 三视角优化报告 + 优化后正文 |
| 想继续做分镜 | 三分钟分镜表 |
| 想继续做 AI 视频 | 礼部TV / LibTV 十五秒版本化母稿 |
| 想检查问题 | 审稿闸门结果 + 退回建议 |

默认优先输出用户能直接看的版本：三分钟正式剧本正文或三分钟剧情框架。除非用户要求，不要把内部审稿表堆在最前面。

## 13 默认调用口令

```text
按通用三分钟剧本工作流，把这一段做成 3 分钟剧本单元；如果输入不足，先自动补齐最低可执行信息；再按题材适配、三分钟标准结构、单集生产卡、正式剧本正文初稿、顶级编剧/导演/剪辑三视角复盘优化、连续性检查、三分钟剧本审稿闸门和通过/退回判定执行。若用户要礼部TV / LibTV / AI视频提示词版本，则在三分钟母稿通过后，继续按顶级编剧、顶级导演、顶级剪辑视角拆成 12 段、每段 15 秒的礼部TV十五秒版本，再进行逻辑性、连续性、因果关系审核并回修十五秒提示词。要求：开场强、目标清楚、压力递进、主角主动选择、结尾有追问；不破坏项目既有世界规则、人物能力和长线因果。
```
