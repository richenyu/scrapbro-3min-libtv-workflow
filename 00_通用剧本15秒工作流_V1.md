# 通用剧本 15 秒工作流 V1

更新时间：2026-06-21

## 用途

这是给任意剧本使用的 LibTV / 里布 15 秒提示词工作流。

它适用于：

```text
任意短剧 / 电影感短片 / 动作戏 / 悬疑戏 / 科幻戏 / 奇幻戏 / 现实戏
任意 Part / 第几段 / 第几个 15 秒
从剧本、总分镜、长提示词、剧情梗概中拆出单条 15 秒视频
生成完整 15 秒提示词、图片绑定卡、缺图补图清单、多格分镜参考图和最终上传包
```

核心原则：

```text
大框架固定，项目细节替换。
流程不写死任何角色、世界观、怪物、道具或场景。
具体剧本的人物名、伙伴名、敌人名、声音文件、锁定图、禁忌项，只在“项目适配层”填写。
```

## 一句话总流程

```text
先冻结版本和来源
-> 用顶级导演 + 顶级编剧 + 顶级剪辑思维预审
-> 直接产出 6-8 个细分镜头草案
-> 确认图片绑定和缺图补图
-> 写连续性四帧和唯一主任务
-> 第一份正式创作输出必须是完整 15 秒 LibTV 提示词
-> 主锁定图全部补齐并 QA 后，重新输出新版完整提示词
-> 再进入多格分镜参考图专项流程
-> 分镜图 QA 通过后回填最终上传顺序
```

## 自动推进硬规则

执行本工作流时，Codex 必须把它当成逐步推进的执行清单，而不是一次性说明文。

```text
第 0 步完成后，自动进入第 0.5 步。
第 0.5 步完成后，自动进入第 1 步。
第 1 步完成后，自动进入第 2 步。
第 2 步完成后：
- 如果缺图，先停在对应补图任务，输出缺图项和补图提示词。
- 如果不缺图，自动进入第 3 步。
第 3 步完成后，自动进入第 4 步。
第 4 步完成后，自动进入第 5 步。
第 5 步完成后，自动进入第 6 步。
第 6 步完成后：
- 如果主锁定图未齐，回到第 2 步补图。
- 如果主锁定图已齐，自动进入第 7 步多格分镜参考图专项流程。
第 7 步完成后，自动进入第 8 步权限边界确认。
第 8 步完成后，自动进入第 9 步分镜图 QA。
第 9 步 QA 通过后，自动进入第 10 步最终交付包。
```

每一步必须输出“本步结果”和“下一步动作”：

```text
本步结果：完成 / 卡住 / 需要补图 / 需要用户确认
下一步动作：继续到第几步，具体要做什么
```

禁止行为：

```text
不能只把完整工作流复述给用户。
不能跳过真实文件和真实图片检查。
不能用“已有”“应该有”代替实际路径确认。
不能在缺图时继续假装完整执行。
不能把多格分镜参考图当普通补图顺手生成。
不能在用户要求执行某段 15 秒时停在流程解释。
```

只有以下情况允许暂停并问用户：

```text
源剧本 / 当前段落无法从文件中确认。
关键锁定图不存在，且无法从现有素材中找到替代来源。
同一段存在多个互相冲突的最新版执行包，文件证据无法判断采用哪版。
用户明确要求先停下来给他看某一步。
```

## 第 0 步：版本冻结卡

任何生成动作前，先列出：

```text
当前剧本 / 项目：
当前段落 / Part / 场次：
当前采用来源稿 / 执行包：
已废弃旧版：
剧情来源总稿：
本段主视觉任务：
分镜格数：
本段主角：
本段伙伴 / 副角色：
本段敌人 / 怪物 / 冲突对象：
当前场景：
当前关键道具：
最终输出目标：
```

如果信息不完整，先查文件；仍不完整，再问用户确认。不能猜。

## 第 0.5 步：顶级导演 / 顶级编剧 / 顶级剪辑预审

版本冻结后，进入图片绑定前，必须先用顶级电影导演、顶级编剧和顶级剪辑师思维优化这一段 15 秒。

目的：

```text
先判断这 15 秒为什么发生、怎么拍、怎么剪、怎么接，再决定图片绑定和完整提示词。
不是机械拆 8 格，不是按剧情文字平均切段。
```

必须先回答：

```text
导演判断：本段最重要的一个画面动作是什么？
编剧判断：本段的因果、人物选择、冲突压力和结果变化是否成立？
信息取舍：哪些信息必须在本段讲清，哪些必须删掉或留到下一段？
开场钩子：前 1-2 秒用什么画面 / 声音把观众抓住？
镜头节奏：本段是追击、急刹、对峙、突袭、反击、取物、揭示、转场还是压迫建立？
剪辑重心：哪些镜头必须快切，哪些镜头必须停半秒让观众看懂？
动作可读性：关键接触点、受力方向、落点是否清楚？
情绪曲线：紧张、缓冲、爆点、余波分别落在哪几秒？
对白功能：如果有对白，是否只推动关系、选择或危机判断，而不是解释过量？
声音剪点：上一段尾音如何进来，本段尾音如何带到下一段？
结尾承接：最后 1-2 秒留给下一段的画面入口或声音入口是什么？
删减判断：哪些信息不能塞进本段，必须留到下一段？
```

预审阶段必须直接输出 6-8 个细分镜头草案：

```text
镜头1｜00:00-00:__：
- 画面 / 动作：
- 编剧功能：
- 剪辑点：
- 声音点：
- 承接关系：

镜头2｜00:__-00:__：
- 画面 / 动作：
- 编剧功能：
- 剪辑点：
- 声音点：
- 承接关系：

继续到 6-8 个镜头，必须覆盖完整 00:00-00:15。
```

硬规则：

```text
必须是 6-8 个镜头，不能少于 6 个，不能超过 8 个，除非用户明确指定。
每个镜头只处理一个清楚动作事件或一个清楚信息点。
时间段必须连续覆盖 00:00-00:15，不能漏秒、重叠或机械平均。
后续完整 LibTV 提示词里的 15 秒细分镜头，必须以这版预审草案为底稿校正和扩写。
如果预审发现本段塞了两个大事件，必须先拆段或删减，再继续。
```

## 第 1 步：图片绑定卡

必须先列出实际上传图片顺序。通用推荐格式：

```text
前置视频节点：
@图片1 场景锁定图：
@图片2 主角锁定图：
@图片3 伙伴 / 副角色 / 对手锁定图：
@图片4 载具 / 武器 / 关键道具锁定图：
@图片5 当前敌人 / 怪物 / Boss / 冲突对象正式锁定图：
@图片6 当前敌人 / 怪物 / Boss 比例图 / 补充图：
@图片7 当前任务道具 / 道具状态图：
@图片8 多格分镜参考图：
@图片9 备用复杂动作 / 特效 / 局部结构参考图：
```

实际项目不必用满 9 张。只上传本段真正出现、真正影响一致性的图片。

图片硬规则：

```text
本段不出现的角色不要上传。
本段不出现的敌人 / 怪物 / Boss 不要上传。
Boss 没正式登场前，不上传 Boss 完整全身图。
完整道具没出现前，不上传完整道具图；只能上传当前状态图或局部状态图。
分镜参考图只能放最后，不能覆盖前面人物、场景、敌人、道具主锁定图。
上一段最后一帧 / 尾帧承接图 / 伪真尾帧图不作为上传图片，不预留图片槽位。
上一段衔接只通过前置视频节点、连续性四帧、声音桥、动作方向、角色状态和场景方向写清楚。
```

图片引用格式硬规则：

```text
最终提示词正文和上传清单必须使用 LibTV 实际上传顺序：@图片1、@图片2、@图片3……连续编号。
不能在最终提示词正文里写 @角色名、@怪物名、@道具英文名 这类素材标签。
每个 @图片N 必须在图片上传顺序里说明中文名称、用途和原文件名 / 锁定内容。
```

## 第 2 步：缺图补图门

如果缺少必要锁定图，必须先补图。补图发生在多格分镜参考图之前。

必须检查：

```text
是否缺主角 / 新人物锁定图：
是否缺伙伴 / 副角色 / 对手锁定图：
是否缺当前场景锁定图：
是否缺当前敌人 / 怪物 / Boss 正式锁定图：
是否缺当前敌人 / 怪物 / Boss 比例图：
是否缺载具 / 武器 / 任务道具锁定图：
是否缺当前道具状态图（残片 / 完整体 / 开启 / 关闭 / 损坏 / 被持有）：
是否缺关键特效 / 局部结构参考图：
```

补图顺序固定：

```text
1. 人物 / 角色锁定图
2. 场景锁定图
3. 敌人 / 怪物 / Boss 正式锁定图
4. 敌人 / 怪物 / Boss 比例图
5. 载具 / 武器 / 任务道具锁定图
6. 道具状态补图
7. 必要的局部特效 / 结构参考图
```

小兵 / 杂兵 / 群体敌人差异化规则：

```text
小兵、杂兵、普通感染者、普通小怪群、路人敌人、群演敌人不能锁成同一张脸。
如果需要补小兵 / 杂兵参考图，只锁共同设计语言：物种方向、感染风格、服装系统、装甲等级、机械感染比例、颜色规则、武器类型等。
每个个体的脸、头骨轮廓、眼位、嘴部、伤口、腐烂纹理、机械残片位置、体型比例、姿态和表情必须不同。
禁止复制同一张脸、同一个头、同一只怪、同一套表情或同一模型排队。
精英怪、Boss、当前关键敌人才允许用单一正式锁定图锁脸、锁比例、锁外形。
```

补图后必须 QA：

```text
人物是否和既有角色一致。
场景是否和本段地点一致。
敌人 / 怪物 / Boss 是否是当前版本，比例是否正确。
如果是群体敌人，是否做到同设计语言但不同脸、不同轮廓、不同细节。
道具状态是否正确。
是否有文字、水印、标题、无关角色。
是否会污染当前段上传槽位。
```

不通过则返工；通过后更新图片绑定卡。

## 第 3 步：连续性四帧

必须写清：

```text
上一段最后一帧：
本段第一帧：
本段最后一帧：
下一段第一帧：
```

这四帧不清楚时，不允许进入正式提示词。

如果这是第一段：

```text
上一段最后一帧：无，直接由场景锁定图 / 主角状态 / 开场动作建立。
本段第一帧：必须写清开场画面和声音钩子。
```

## 第 4 步：唯一主任务

每个 15 秒只允许一个主视觉任务。

必须写成一句话：

```text
本段只解决：……
```

禁止在一段里同时完成两个大事件。下一段动作不能提前写进来。

## 第 5 步：第一份正式输出必须是完整 15 秒提示词

版本冻结卡、导演 / 编剧 / 剪辑预审卡、图片绑定卡、缺图补图门和连续性四帧确认后，第一份正式创作输出必须是：

```text
按 LibTV / 里布提示词写作方式优化后的完整 15 秒提示词。
```

不能先输出分镜参考图提示词，不能先生成多格分镜图，不能只写剧情梗概。

完整提示词必须包含：

```text
图片上传顺序
本段唯一主视觉任务
人物一致性
场景一致性
敌人 / 怪物 / Boss 一致性
道具一致性
声音一致性 / 角色声音锁定
字幕规则
动作地理 / 空间轴线
15 秒细分镜头
结尾承接
防错审计
```

## 第 6 步：一致性和声音锁定必须显式写入

正式提示词和分镜参考图提示词中，必须同时包含：

```text
人物一致性：
- 主角 / 伙伴 / 关键角色的脸、发型、服装、身体比例、装备、伤损状态、手持道具、站位关系。
- 禁止换脸、换装、无理由变比例、无理由互换位置。

场景一致性：
- 当前场景锁定图名称。
- 时间、地点、光线、地面材质、核心地标、可出现物件、禁止物件。
- 可以切机位，不能突然跳地点或跳时代。

敌人 / 怪物 / Boss 一致性：
- 当前正式锁定图和比例图。
- 体型等级、攻击方式、移动方式、弱点、禁止跑偏方向。
- Boss 未登场时不提前完整露脸；小兵群必须同设计语言但个体不同。

道具一致性：
- 当前道具状态、位置、完整度、开合状态、在谁手里。
- 残片就是残片，完整体就是完整体；不能混。

声音锁定：
- 本段所有发声角色必须单独列出。
- 每个发声角色必须写清声线来源 / 声音锁定文件、年龄感、音色、语速、情绪状态、机械处理或空间处理比例、混音关系。
- 没有对白但有呼吸、短笑、喘息、电子提示音、扫描声、通讯声、低频启动音、怪物嘶吼或环境触发声的对象，也必须列入声音锁定。
- 声音必须像真实现场收音，不要旁白感、字幕朗读感、AI 播报感或录音棚独白感。
- 环境音和动作声可以大于对白，但不能盖住关键信息。
- 上一段尾音如何进入本段、本段尾音如何接下一段必须写清楚。
```

字幕规则：

```text
有对白时，按当前项目语言规则执行。
如果项目要求英文配音：实际配音只读英文对白，中文翻译只供理解，不配音、不生成字幕、不出现在画面里。
如果项目要求中文配音：只保留中文对白，不额外生成说明字幕。
无论哪种语言，禁止标题字幕、说明性字幕、弹幕字幕、水印和对白气泡。
```

## 第 7 步：多格分镜参考图专项流程

多格分镜参考图不是普通补图，不能在缺图补图门里顺手生成。

进入条件：

```text
人物 / 角色锁定图已确认。
场景锁定图已确认。
敌人 / 怪物 / Boss 正式锁定图已确认。
比例图 / 补充图已确认。
载具 / 武器 / 任务道具锁定图已确认。
道具状态图已确认。
声音锁定和字幕规则已写入完整提示词。
连续性四帧已确认。
所有缺图已补齐并通过 QA。
图片绑定卡已更新为真实上传顺序。
```

专项流程：

```text
1. 重新输出“新版本完整 15 秒提示词”。
   - 新版提示词必须整合所有已确认锁定图、最新 @图片N 顺序、连续性四帧、声音锁定和动作地理。
   - 不能拿补图前的旧提示词直接生成分镜参考图。

2. 写“多格分镜参考图一致性生图提示词”。
   - 逐张说明输入参考图：哪张锁场景、哪张锁主角、哪张锁伙伴、哪张锁载具、哪张锁敌人 / 怪物 / Boss、哪张锁比例、哪张锁道具。
   - 写清每一格对应新版完整提示词里的哪个时间段和镜头事件。
   - 写清分镜图只能学习动作顺序、镜头节奏、站位、方向、起点、落点和结尾承接。
   - 写清分镜图不能改写人物脸、伙伴结构、载具结构、敌人物种、真实比例、道具状态和场景地标。

3. 实际把所有主锁定图作为参考输入放进生图工具。
   - 不能只用文字生成正式多格分镜参考图。
   - 如果工具不支持参考图输入，生成结果只能算临时草图，不能进入最终上传图包。

4. 按新版完整提示词生成对应格数多格分镜参考图。
   - 6 个细分镜头生成 6 格。
   - 7 个细分镜头生成 7 格。
   - 8 个细分镜头生成 8 格。
   - 禁止把 7-8 个镜头压缩成 6 格。

5. 生成后进入分镜图 QA。
   - QA 通过后，多格分镜参考图才能作为最后一张 @图片N 回填。
   - QA 不通过，错误图必须隔离，不得作为下一版输入参考。
```

## 第 8 步：分镜图权限边界

分镜图只能锁：

```text
动作顺序
镜头节奏
站位
方向
起点和落点
结尾承接
```

分镜图不能锁：

```text
人物脸
伙伴 / 副角色的特殊结构
载具结构
敌人 / 怪物 / Boss 物种
真实比例
道具状态
场景地标
```

这些必须以前面的正式锁定图为准。

## 第 9 步：分镜图生成后 QA

必须检查：

```text
是否对应正确格数。
是否一格对应一个细分镜头。
是否有字幕、标题、水印、对白气泡。
是否复制角色。
是否把普通敌人变成 Boss，或把 Boss 弱化成普通敌人。
是否让分镜图覆盖了人物 / 场景 / 敌人 / 道具锁定。
是否把旧版本角色、旧敌人、旧场景带回来。
小兵 / 杂兵群是否出现同一张脸、同一个头、同一套表情复制。
结尾是否能自然接下一段。
```

不通过则返工，不允许直接回填。

## 第 10 步：最终交付包

最终交付必须包含：

```text
最终上传图包路径
可复制提示词文件
图片1-N 上传顺序
当前采用版本
已废弃旧版
防错审计
分镜图权限说明
所有真实存在图片的本地链接和预览
```

## 固定输出结构

以后每条完整 15 秒提示词，按这个结构输出：

```text

```

## 项目适配层

通用工作流执行到具体剧本时，必须把以下槽位替换为当前项目自己的设定：

```text
项目名称：
世界观风格：
主角锁定：
伙伴 / 副角色锁定：
敌人 / 怪物 / Boss 锁定：
场景美术规则：
道具状态规则：
声音锁定文件：
对白语言规则：
禁止跑偏项：
图片资产目录：
当前段落执行包：
```

项目适配层可以包含很具体的硬规则，但不能反向改掉通用主流程。
