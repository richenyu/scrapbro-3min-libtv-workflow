# 33_通用礼部TV提示词逻辑连续因果审查工作流

更新时间：2026-06-21
版本：V2.2｜礼部TV / LibTV 提示词逻辑、连续性、因果审查闸门｜因果矩阵与状态账版

## 文件定位

本文件是四步制作链路中的第三步。

```text
1. 三分钟剧本工作流
-> 2. 三分钟剧本转礼部TV提示词工作流
-> 3. 礼部TV提示词逻辑 / 连续性 / 因果审查工作流
-> 4. 十五秒礼部提示词工作流
```

本工作流只做审查闸门和回修任务单。

它不直接替代三分钟剧本工作流。
它不直接替代三分钟转礼部TV提示词工作流。
它不直接替代 15 秒礼部提示词生成 / 精修工作流。
它不直接生成分镜参考图。

V2.1 优化重点：

```text
审查模式可选择，避免每次都全量重审。
问题定位必须精确到 EP 时间码 + Part + 镜头号。
回修任务单必须能直接交给第四步执行。
明确什么时候不能交给第四步，必须退回第二步。
缺图 / 锁图加入阻断规则。
对白不只检查位置，还要检查能不能在对应秒数内生成。
审查报告开头先给可执行摘要。
```

V2.2 优化重点：

```text
新增因果链矩阵，逐 Part 检查：触发 -> 目标 -> 限制 -> 选择 -> 动作 -> 代价 -> 结果 -> 状态变化 -> 下一步原因。
新增状态变化账，强制记录每个 Part 前后的人物、位置、道具、伤势、信息、敌人收益和声音状态。
新增信息连续性审查，检查角色知道什么、不知道什么、什么时候知道、为什么知道。
新增结果变化门，任何动作必须改变位置、状态、道具、信息、关系或风险，否则判定为装饰动作。
新增反向验证，从结果倒推原因，防止看起来顺但因果缺环。
新增修复原则，要求最小改动、近因修复、可见动作修复，禁止用旁白硬补逻辑。
```

## 01 核心目标

把已经由第二步输出的礼部TV / LibTV 完整提示词包，审成可进入第四步逐段精修的状态。

核心问题不是“写得美不美”，而是：

```text
整集因果是否成立
12 段是否连续
每段 15 秒内部是否能拍完
每个细分镜头是否有起点、接触点、结果
对白是否放在正确镜头里
人物、场景、声音、道具、伤势、情绪是否连续
图片引用、锁定图和缺图补图是否会导致生成跑偏
```

```text
本集最后一帧：
下一集第一帧入口：

场景空间表：
道具 / 武器 / 伤势状态表：
信息状态表：

声音一致性规则：
图片资产与上传顺序：
缺图补图清单：
```

如果缺少来源礼部TV提示词包，不进入本工作流，先回第二步。
如果缺少上一段 / 下一段承接状态，先补状态表，再审。

## 02.5 审查模式选择

进入审查前，先选择本次审查模式。

```text
全集审查：
检查整集 12 段提示词包，适合第二步刚完成后第一次进入第三步。

单 Part 审查：
只审某一个 15 秒 Part，适合第四步返修后复查。

多集连续审查：
检查 EP01 -> EP02 -> EP03 的集尾 / 集首承接，适合连续剧批量制作。

缺图与锁图专项审查：
只审图片资产、上传顺序、角色锁图、场景锁图、敌人锁图、道具锁图和缺图阻断。

对白与声音专项审查：
只审对白是否嵌入镜头、英文台词是否过长、声音情绪是否连续、是否会生成错误字幕。
```

如果用户没有指定模式，默认使用：

```text
全集审查 -> 输出总问题清单 -> 再按问题严重度进入单 Part 复查。
```

## 03 总流程

```text
读取输入包
-> 选择审查模式
-> 冻结来源三分钟剧本和礼部TV提示词包
-> 建整集因果链
-> 建因果链矩阵
-> 建 12 段连续性表
-> 建每段 15 秒内部镜头链
-> 建人物 / 场景 / 道具 / 声音 / 图片状态账
-> 建信息状态账和敌人收益账
-> 过十二道审查门
-> 按 S/A/B/C 分级
-> 每个问题定位到 EP 时间码 + Part + 镜头号
-> 输出证据链
-> 输出可执行摘要
-> 输出回修任务单
-> 判定退回哪一步
-> 交给第四步 15 秒工作流逐段修
```

## 03.5 问题定位硬规则

所有问题必须定位到具体时间和镜头，不能只写 Part 编号。

标准定位格式：

```text
EP01 00:45-01:00｜Part04｜镜头3｜Part 内 00:05-00:07｜2秒
```

如果原提示词没有写镜头号或秒数，先标注：

```text
定位失败：原提示词缺少镜头号 / 秒数。
处理：退回第二步补齐时间码，或列为 A 级问题交给第四步先补镜头结构。
```

## 03.6 因果链矩阵

审查时必须先建立因果链矩阵。每个 Part 至少填一行。

```text
Part：
触发事件：
主角此刻目标：
外部限制：
内部限制：
可选方案：
实际选择：
可见动作：
付出代价：
直接结果：
状态变化：
下一 Part 为什么必须发生：
```

硬规则：

```text
任一关键格为空，至少判 B 级。
目标不清、限制不清、选择不清，至少判 A 级。
只有动作，没有代价或状态变化，判 B 级；影响主线时判 A 级。
下一 Part 不能由上一 Part 的结果自然推出，判 A 级连续性问题。
```

## 03.7 状态变化账

每个 Part 都必须记录前后状态，不能只写“继续逃、继续打、气氛升级”。

```text
Part：
开始位置：
结束位置：
人物开始状态：
人物结束状态：
道具开始归属：
道具结束归属：
伤势 / 污渍 / 服装变化：
已知信息变化：
敌人收益 / 损失：
声音状态变化：
下一段必须继承：
```

状态变化规则：

```text
上一段结束状态必须成为下一段开始状态。
伤势、血迹、灰尘、湿痕、破损不能自动消失。
道具不能无来源出现，也不能无动作换手。
角色不能知道未听见、未看见、未被告知的信息。
敌人不能无理由知道主角位置，也不能无理由停止追击。
```

## 04 十二道审查门

### 4.1 整集大因果门

检查：

```text
上一集结果是否自然引出本集开场。
本集开场是否逼出本集小目标。
本集小目标是否贯穿 12 段。
每个关键事件是否有：原因 -> 限制 -> 选择 -> 动作 -> 代价 -> 结果。
本集小答案是否来自前面铺垫。
结尾钩子是否从本集结果自然长出来。
```

### 4.2 12 段连续性门

逐段检查：

```text
Part01 最后一帧 -> Part02 第一帧
Part02 最后一帧 -> Part03 第一帧
继续到 Part12
```

每次承接都必须写清：

```text
画面停在哪里
人物站在哪里
敌人 / 危险在哪里
道具和伤势是什么状态
声音如何衔接
下一段为什么从这里开始
```

### 4.3 每段 15 秒内部逻辑门

每个 Part 内部检查 6-8 个细分镜头：

```text
镜头1 为什么能接镜头2
镜头2 为什么能接镜头3
动作起点在哪里
谁先动
谁反应
接触点是什么
动作结果落在哪里
镜头结束时状态有没有改变
```

禁止只写“激烈打斗、快速闪避、气氛压迫、镜头推进”。

### 4.4 对白嵌入与可生成性门

检查每一句对白是否放在具体发生的镜头里。

合格格式：

```text
镜头4｜EP01 00:05-00:07｜Part02 00:05-00:07｜2秒：
   - 对白 / 字幕：角色名: "English line."（中文旁译：……）
```

不合格：

```text
把对白统一放在段落底部。
镜头里没有对白字段。
对白发生时间不清楚。
一个角色在无法说话、奔跑、坠落、被压制时说长句。
```

同时检查对白是否能生成：

```text
这句英文对白能否在对应秒数内说完。
角色此刻是否有呼吸、站位、口型和情绪条件说话。
动作强度是否会压过对白。
对白是否会导致 AI 生成中文字幕、说明字幕或对白气泡。
同一 15 秒内是否出现多人连续长对白。
```

对白长度参考：

```text
1-2 秒：最多 3-5 个英文词。
2-3 秒：最多 6-9 个英文词。
3-5 秒：最多 10-14 个英文词。
动作戏 / 坠落 / 奔跑 / 被压制时，只允许短句、气声、喊叫或无对白。
```

### 4.5 人物状态连续门

检查：

```text
脸、年龄、体型、服装、发型是否连续。
伤口、血迹、灰尘、湿痕、裂纹是否连续。
武器和道具是否还在正确的人手里。
情绪是否从上一镜头自然过渡。
声音年龄感、语速、情绪是否一致。
```

### 4.6 场景空间连续门

检查：

```text
入口、出口、危险来源、人物左/右/前/后是否清楚。
镜头方向是否突然反转。
角色是否瞬移到另一个位置。
上一镜头的障碍、门、墙、台阶、桥、车、塔、笼子是否消失。
```

### 4.7 动作物理因果门

每个关键动作必须能回答：

```text
谁发力？
从哪里发力？
打到哪里？
对方为什么被击退 / 被压住 / 被救出 / 被挡住？
动作结果改变了什么：位置、状态、道具、信息、关系、反派收益？
```

如果动作只让画面更帅，但不改变任何状态，判为 B 级或 A 级问题。

### 4.8 图片资产与锁定图门

检查：

```text
每段上传顺序是否清楚。
@图片1、@图片2、@图片3 是否和上传清单一致。
人物锁定图、场景锁定图、敌人 / Boss 锁定图是否齐全。
缺图是否已经列入补图清单。
是否有错误上传导致角色污染或场景污染。
```

缺核心锁定图时，不允许进入分镜图生成。

缺图阻断规则：

```text
缺主角锁定图：不能判定为最终可生成版，只能进入提示词草稿。
缺核心场景图：不能进入分镜参考图生成，先补场景图。
缺关键敌人 / Boss 图：敌人容易变脸或变体，必须补图后再出最终版。
缺关键道具图：道具会影响动作因果时，必须补图或在第四步重写为无道具方案。
上传顺序不清：不能进入第四步最终输出，只能先补图片上传顺序。
```

### 4.9 生成风险门

检查：

```text
15 秒是否拍得完。
是否一个 Part 塞了两个大事件。
是否复杂特效过多。
是否多人对白过密。
是否镜头太抽象导致 LibTV 只会生成氛围画面。
是否存在平台容易生成标题、水印、中文字幕、说明字幕的问题。
```

### 4.10 信息连续性门

检查每个角色在每个 Part 里“知道什么”。

```text
主角知道了什么？
主角不知道什么？
敌人知道了什么？
旁观者 / 同伴知道了什么？
这条信息从哪里来：看见、听见、触碰、被告知、推理、旧记忆？
知道以后是否改变了选择？
不知道的信息是否被角色错误使用？
```

常见硬伤：

```text
角色使用了自己不可能知道的信息。
敌人提前预判主角动作，但没有侦察、线索或能力来源。
主角突然理解机关 / 法术 / 地图，但前面没有任何可见线索。
角色忘记刚刚发生的重要伤害、警告、约定或代价。
```

### 4.11 结果变化门

每个关键动作都必须改变至少一种状态。

```text
位置变化
身体状态变化
道具归属变化
信息状态变化
关系变化
危险等级变化
敌人收益 / 损失变化
下一段行动条件变化
```

如果一个动作只让画面更帅，但不改变任何状态：

```text
删掉。
合并到其他镜头。
改成能制造状态变化的动作。
或标为 B / A 级问题交给第四步重写。
```

### 4.12 反向验证门

审查完正向因果后，必须从结果倒推一次。

```text
本集结尾为什么一定会发生？
本集小答案靠哪几个前置动作支撑？
关键道具为什么会在此刻出现？
关键伤势为什么会在此刻影响选择？
反派为什么此刻获得优势或失去优势？
下一集入口是否只能由本集结果推出？
```

如果倒推找不到前因，说明正向叙述看似顺，但实际缺因。

## 05 S/A/B/C 分级

| 等级 | 定义 | 处理 |
| --- | --- | --- |
| S 级 | 整集大因果断裂、故事结果错、角色动机崩、关键动作不可能、结尾无法由前文推出。 | 退回第一步三分钟剧本工作流或第二步重出礼部TV提示词包。 |
| A 级 | 12 段承接断、人物状态断、信息状态断、关键镜头动作因果缺失、对白位置严重错误。 | 必须修，通常退回第二步或交给第四步重点修。 |
| B 级 | 局部镜头不够清楚、动作接触点弱、状态账漏记、图片顺序需补、装饰动作没有结果变化。 | 交给第四步 15 秒工作流逐段修。 |
| C 级 | 表达可更清楚，不影响核心生成。 | 第四步顺手润色。 |

## 06 判定退回哪一步

```text
S 级大因果问题 -> 退回第 1 步三分钟剧本工作流。
S / A 级礼部TV转写结构问题 -> 退回第 2 步三分钟剧本转礼部TV提示词工作流。
A / B 级单段提示词问题 -> 进入第 4 步十五秒礼部提示词工作流逐段修。
C 级表达问题 -> 第 4 步中顺手处理。
```

必须退回第二步的硬门槛：

```text
超过 3 个 Part 出现结构性承接断裂。
超过 3 个 Part 缺少完整 6-8 个细分镜头。
超过 3 个 Part 没有秒数或镜头号，无法精确定位问题。
整集图片上传顺序混乱，导致多数 Part 无法判断 @图片 对应关系。
第二步输出只像剧情摘要，不是完整礼部TV提示词包。
人物 / 场景 / 声音一致性没有建立，第四步无法逐段修。
因果链矩阵无法填完整，超过 3 个 Part 缺目标、限制、选择或结果。
状态变化账无法建立，超过 3 个 Part 无法继承上一段状态。
```

第三步不直接把所有 Part 改成最终提示词。
第三步只给出清楚的问题、证据和回修指令。

## 06.5 修复原则

第三步只开任务单，但必须指定修复原则。

```text
最小改动：只修断掉的因果、状态或镜头链，不重写整段风格。
近因修复：缺哪个原因，就在最近的前一镜头或前一 Part 补原因。
可见修复：优先用动作、视线、道具、声音、伤势变化修逻辑，不用旁白解释硬补。
结果保留：如果剧情结果已通过，不得让第四步随意改变结果。
代价保留：已经建立的伤势、损失、暴露、敌人收益不能被修没。
信息保留：角色不知道的信息不能因为修稿而突然知道。
不新增大设定：第四步不能为了修一个镜头新增未铺垫能力、道具或世界规则。
```

禁止修法：

```text
用一句旁白解释本来该拍出来的动作。
用“突然、原来、其实、刚好”修因果。
把角色能力临时加强来跳过限制。
让敌人突然变笨来让主角过关。
为了连续性删掉已经承诺给下一集的钩子。
```

## 08 回修任务单写法

每个问题必须给到能执行的修法。

合格：

```text
Part04｜A级｜动作因果断
精确定位：EP01 00:45-01:00｜Part04｜镜头3-5｜Part 内 00:05-00:10
证据：镜头3 写“她被卫兵按住”，镜头4 直接写“她翻身夺弓”，中间没有脱困动作。
必须修：补一个 1-2 秒动作镜头，写清她用膝盖撞开谁、哪只手先挣脱、弓从谁手里转移到她手里。
保留什么：保留 Part04 的剧情结果，她必须拿到弓并继续逃。
不能改什么：不能改成别人主动把弓递给她，不能新增未铺垫能力。
按什么修复原则：近因修复 + 可见修复 + 结果保留。
必须新增什么镜头：新增镜头4A，1-2 秒，写清脱困动作起点、接触点、结果。
必须删除什么空话：删除“快速挣脱”“激烈反击”这类不可拍概括。
第四步重写范围：只重写 Part04 镜头3-5。
交给第4步：用 15 秒工作流重写 Part04 的镜头3-5，保留原剧情结果。
```

不合格：

```text
Part04 逻辑有问题，建议加强。
```

## 09 通过标准

```text
没有 S 级问题。
A 级问题已经明确退回或列入第四步必修。
没有触发“必须退回第二步”的硬门槛。

每句对白都在具体镜头里。
每句对白都能在对应秒数内生成。

缺图补图清单明确。
缺图阻断项已经标清：草稿可继续 / 最终生成阻断 / 分镜图阻断。
生成风险已经分配到具体 Part。
因果链矩阵已经填完，关键格没有空缺。

```
